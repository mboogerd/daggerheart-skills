# Daggerheart Adversary Selection

Use this skill when the user knows the scene but not which kinds of enemies belong in it.

## Workflow

1. Read `references/core-procedure.md`.
2. Read `references/role-pairings.md` if the encounter needs stronger synergy.
3. Read `references/examples.md` for role-mix models.
4. Read `references/edge-cases.md` if the encounter is overloaded, too flat, or too grindy.
5. Use `assets/template.md` for a structured encounter role brief.
6. Choose the pressure each adversary role should apply.
7. Build a mix that creates different player decisions.
8. Hand off to `daggerheart-adversary-creation` or `daggerheart-adversary-scaling` if specific stat blocks are needed.

## Output

- Recommended roles
- Why each role is present
- What player behavior each role pressures
- Which role is the encounter's backbone
