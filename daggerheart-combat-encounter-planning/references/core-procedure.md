# Core Procedure

## Narrative First

Build the encounter from the question: what does this fight reveal, pressure, or change?

Start with:

- the encounter's narrative function
- the adversaries' motives
- the battlefield's active pressures

Do not start from stat blocks alone.

## Budget Second

Use battle points as rough balancing help, not as a guarantee of fairness.

Starting battle points:

`(3 x number of PCs) + 2`

Common adjustments:

- subtract 1 for a shorter or easier fight
- add 2 for a longer or more dangerous fight
- subtract 2 if using 2 or more solos
- subtract 2 if all adversaries deal extra damage
- add 1 if using lower-tier adversaries
- add 1 if the encounter has no bruiser, horde, leader, or solo

Spend rates:

- minion group equal to party size: 1
- social or support: 1
- horde, ranged, skulk, standard: 2
- leader: 3
- bruiser: 4
- solo: 5

## Composition Rule

The encounter should ask more than one tactical question.

Examples:

- hold the line while ranged pressure punishes open ground
- break the leader-support engine before the bruiser overwhelms melee
- stop the ritual while navigating a hostile environment

## Environment Rule

An encounter environment should change player decisions, not just decorate the scene.

If the battlefield is intentionally plain, say so and explain why.

## Delegation Rule

This skill plans the encounter and marks what must happen next.

It should not:

- write full adversary stat blocks unless the user explicitly collapses planning and creation into one request
- replace environment-design work with a half-finished environment block
- ignore likely reuse of existing adversaries when lookup or adaptation is the better fit
