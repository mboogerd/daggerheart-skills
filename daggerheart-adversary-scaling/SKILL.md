# Daggerheart Adversary Scaling

Use this skill when an adversary concept fits but the numbers do not.

## Workflow

1. Read `references/core-procedure.md`.
2. Read `references/examples.md` for before-and-after scaling examples.
3. Read `references/edge-cases.md` if the move crosses several tiers or touches solos, hordes, or control-heavy features.
4. Use `assets/template.md` for a scaling report.
5. Replace tier-based numbers first.
6. Adjust special features second.
7. Check Hordes, Minions, and Solos separately.

## Output

- New tier
- Replaced stat line
- Feature changes
- Any caution about edge cases
